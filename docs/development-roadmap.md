# 开发任务流程大纲

## 阶段 1：项目骨架

状态：已开始实施。

目标：

- 建立前端、后端、Agent、知识库和文档目录。
- 提供本地开发配置和最小可运行入口。
- 加入基础测试样例。

交付：

- React + TypeScript + Vite 前端骨架。
- FastAPI 后端骨架。
- LangGraph/LangChain 依赖声明。
- MySQL 配置占位。
- Markdown/HTML 知识库目录。

## 阶段 2：前后端聊天联通

状态：已升级为 SSE 流式输出。

确认点：

- 前端默认调用 `/api/chat/stream`。
- 后端保留 `/api/chat` 普通 REST 接口用于兼容。

计划：

- 前端发送消息到 `/api/chat/stream`。
- 后端通过 SSE 返回 `status`、`token`、`done` 事件。
- 前端逐 token 追加 AI 回复，并展示错误和来源信息。

## 阶段 3：MySQL 玩家数据工具

状态：已实现玩家基础资料查询工具；充值记录和封禁状态查询仍待表结构确认后补充。

确认点：

- 玩家表、充值表、封禁表或现有游戏 API 的字段结构。
- 查询权限和脱敏规则。

计划：

- 创建只读 MySQL 连接。
- 实现受控工具函数。
- 添加数据库工具测试。
- 禁止模型直接生成 SQL。

## 阶段 4：Markdown/HTML 知识库检索

状态：已实现轻量关键词检索版本。

确认点：

- 知识库文件来源、目录结构和更新方式。
- 本地向量库用 Chroma 还是 FAISS。

计划：

- 解析 Markdown/HTML。
- 清洗正文并按标题分块。
- 建立检索索引。
- 回复时返回引用来源。

## 阶段 5：Agent 流程编排

状态：已实现 LangGraph 基础状态图。

已补充：

- DeepSeek/OpenAI-compatible LLM 决策节点。
- 模型动作 JSON 解析和非法输出回退。
- 工具执行后再由模型总结最终回复。
- LLM 未启用时继续使用规则流程。
- 内存级会话记忆，按 `session_id` 保留最近 10 条对话。

确认点：

- 哪些问题必须转人工。
- 哪些玩家数据可以直接回复，哪些必须脱敏。

计划：

- 使用 LangGraph 编排分类、工具调用、回复生成和安全检查。
- 支持玩家数据问题、知识库问题和混合问题。
- 保存关键审计日志。

## 阶段 6：验证与本地运行

状态：安全与客服策略已先行实现；本地运行脚本已补齐。

已实现：

- 拒答系统提示词、API key、密钥等内部信息请求。
- 退款、投诉、申诉、人工客服诉求转人工。
- 回复中脱敏手机号和身份证号。
- 知识库无结果时返回补充信息兜底。
- PowerShell 脚本启动后端和前端。
- 本地环境检查命令。

确认点：

- 是否需要 Docker Compose 管理 MySQL 和服务启动。

计划：

- 增加后端接口测试。
- 增加前端组件测试。
- 增加 Agent 流程测试。
- 补齐本地运行说明。
